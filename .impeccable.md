# Design Context ‚Äî TIB Control Surface

## Target Audience
Single operator (Marouane) running a personal AI-operated media company. Power user. Always logged in. Uses this dashboard daily to monitor the entire MIMULE/TechInsiderBytes stack across multiple services.

## Use Cases
- Monitor the editorial autopipeline (queue depth, stage progress, failures)
- Observe LiteLLM model costs, latency, and health across 130+ models
- Triage incidents and receive alerts
- Inspect finance intelligence runs and scout topic shortlists
- Manage builder workflows (AI code agents)
- Review governance policies and audit trails
- Track article production (NewsBites editorial pipeline)

## Brand Personality / Tone
**Operator terminal, not consumer app.** Dense, precise, information-first. Think Bloomberg terminal meets GitHub Actions ‚Äî dark-capable, monospace numbers, status badges over descriptions. The operator trusts the data and wants it all visible without clicking through menus. No marketing fluff, no empty states with cheerful illustrations. When something is wrong, it should be visually alarming. When things are fine, quiet confidence.

## Design Principles
1. **Density over whitespace** ‚Äî tables should show 20-50 rows with pagination, not 5-card layouts
2. **Scannable** ‚Äî status badges (color-coded), monospace for IDs/hashes/costs, timestamps relative
3. **Progressive disclosure** ‚Äî data-heavy sections (100+ rows) collapsed by default; user expands what they need
4. **Interactivity** ‚Äî every table column header is clickable to sort; every table has a search/filter input; row counts shown
5. **No modal overuse** ‚Äî inline expand preferred over dialogs for reading; dialogs only for destructive actions
6. **Consistent chrome** ‚Äî same `.page` wrapper, same `.page-header` pattern, same `.stat-card` grid at top

## Anti-Patterns to Avoid
- Tables with no sorting or filtering (the main current issue)
- SectionCard with `defaultOpen=true` when the section has more than ~10 rows
- Hardcoded `height` or `max-height` on scroll containers that cut off content unexpectedly
- Generic loading spinners with no skeleton states for known data shapes
- Empty arrays rendered as blank space instead of "No data" cells
- DaisyUI class contamination (`btn`, `badge`, `card`, `modal`) ‚Äî use project CSS classes only
- Inline `style={{}}` blocks for colors that should be CSS tokens
- `alert()` / `confirm()` native dialogs ‚Äî use ConfirmModal component

## CSS Token Reference
Primary colors: `var(--accent)`, `var(--bg-panel)`, `var(--border)`, `var(--text-muted)`
Status: `.status-badge.success`, `.status-badge.error`, `.status-badge.warning`, `.status-badge.info`
Layout: `.page`, `.page-header`, `.stat-card`, `.data-table`, `.table-container`
Typography: `.mono` for IDs/hashes, `.text-muted` for secondary info
