# Data Processing Agreement (DPA)

**Version**: 1.0.0  
**Effective**: 2026-05-17

---

## Overview

This Data Processing Agreement ("DPA") forms part of the Terms of Service between the Controller (TechInsiderBytes, "TIB") and the Customer ("Data Subject" or "Customer") when processing personal data in connection with the Builder Platform.

---

## Roles

| Party | Role | Responsibility |
|---|---|---|
| **TechInsiderBytes** | Data Processor | Operates the Builder Platform, processes data on behalf of the Customer |
| **Customer** | Data Controller | Determines purposes and means of processing; owns their data |

TIB acts as a data processor under Article 28 of GDPR. TIB does not own or control Customer data — the Customer retains full ownership and control.

---

## Scope of Processing

### Subject Matter
The processing concerns the operational data generated by the Customer's use of the Builder Platform — including workflow definitions, run logs, audit entries, and system telemetry.

### Nature
Processing is performed by automated systems (AI agents, workflows, diagnostic tools). No human operators access Customer data except when specifically requested by the Customer or for maintenance purposes.

### Purpose
- Execute Customer-defined workflows
- Generate and store workflow artifacts
- Produce audit logs for compliance
- Provide diagnostic and monitoring services
- Improve platform reliability and performance

### Duration
Processing continues for the duration of the service subscription. Upon termination, data is retained per the Data Retention policy and deleted within 90 days.

---

## Data Categories

| Category | Examples | Retention |
|---|---|---|
| Workflow definitions | YAML files, agent prompts, validation rules | Duration of subscription + 90 days |
| Run logs | Pass output, validation results, artifacts | 30 days (configurable up to 365) |
| Audit entries | All API calls, auth events, changes | 365 days (immutable) |
| Telemetry | Aggregate platform metrics | 30 days (no personal data) |
| Secrets | Encrypted credentials (values never exposed) | Until deleted by Customer |

**No special category data (Article 9 GDPR) is processed.**

---

## Sub-Processors

TIB uses the following sub-processors:

| Sub-processor | Purpose | Country | Adequacy |
|---|---|---|---|
| Hetzner Cloud | Virtual server hosting | Germany | EU (Adequacy Decision) |
| Cloudflare | CDN and reverse proxy | US | EU-US Data Privacy Framework |
| OpenRouter | AI model inference (fallback) | US | EU-US Data Privacy Framework |
| GitHub | Version control and CI | US | EU-US Data Privacy Framework |

Sub-processors are only used for tasks specified by TIB. They are contractually bound to GDPR-compliant data handling.

---

## Security Controls

See [Security Overview](../compliance/security-overview.md) for full details of technical and organizational measures.

Key controls:
- TLS 1.2+ for all data in transit
- AES-256 for data at rest (database, artifacts)
- Role-Based Access Control (RBAC)
- Audit chain (append-only, cryptographically chained)
- Secrets encryption (values never logged or returned via API)
- Rate limiting on all endpoints

---

## Data Subject Rights

Customers may exercise the following rights regarding their data:

| Right | How to exercise |
|---|---|
| Access | `GET /api/governance/secrets` (keys only), `GET /api/audit/export` |
| Rectification | Update workflow YAML, re-run affected workflows |
| Erasure | Delete workflows, artifacts, workflows via API or UI |
| Portability | Export via `GET /api/audit/export` in JSON format |
| Restriction | Contact support to restrict processing |
| Objection | Contact support to object to specific processing |
| Human review | Contact support for automated decision appeals |

Requests can be submitted to `privacy@tib.com`. TIB responds within 30 days.

---

## Data Breaches

In the event of a data breach:
1. TIB notifies Customer within 72 hours of becoming aware
2. Notification includes nature of breach, data affected, remedial steps
3. Customer is kept informed until resolution

No payment is required for breach notification.

---

## Data Residency

All data is stored and processed in the **European Union** (`eu-west-1`, Ireland). No data is transferred outside the EU except via the sub-processors listed above, all of which are covered by EU adequacy decisions or the EU-US Data Privacy Framework.

---

## Onward Transfers

Data may only be transferred to third countries in accordance with:
- EU adequacy decisions
- Standard Contractual Clauses (SCCs)
- Binding Corporate Rules (BCRs)

TIB does not use SCCs or BCRs as primary transfer mechanisms — all transfers are covered by adequacy decisions where applicable.
